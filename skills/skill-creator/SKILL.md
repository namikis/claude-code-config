# Skill Creator

Claude CodeのSkillを作成・改善するスキル。

## ワークフロー

### Step 1: 要件ヒアリング

ユーザーの指示から以下を特定する。不明確な点は作成前に必ず確認する：

| 項目 | 確認内容 | デフォルト |
|------|----------|------------|
| 目的 | スキルが何をするか | — （必須） |
| トリガー条件 | どんな指示・状況で起動すべきか | ユーザーの指示から推測 |
| 出力 | 期待される振る舞い・成果物 | — （必須） |
| スコープ | ユーザー or プロジェクト | 確認する |
| 複雑度 | 単純（SKILL.mdのみ）or 複合（references/等を使用） | 指示内容から判断 |

**特に確認すべき場面：**
- スコープの指定がない場合（ユーザー `~/.claude/skills/` vs プロジェクト `.claude/skills/`）
- トリガー条件が曖昧で複数解釈が可能な場合
- 副作用を伴う操作（デプロイ、DB変更等）で `disable-model-invocation` の要否判断が必要な場合

### Step 2: SKILL.mdの設計・作成

`references/skill-design-guide.md` を参照し、以下の手順で作成する：

1. **フロントマター設計**: description最適化が最重要。Claudeはdescriptionをセマンティックマッチングで自動起動判定に使うため、具体的なトリガーフレーズを複数含めないと起動漏れが発生する
2. **本文設計**: Progressive Disclosureを意識し、SKILL.md本文は手順の流れに集中。詳細知識はreferences/に外部化する。SKILL.md本文はスキル起動のたびに全文ロードされるため、肥大化するとコンテキストを圧迫する
3. **ディレクトリ構成決定**: 単純なスキルはSKILL.mdのみで十分。補助ファイルを増やすとClaudeがファイル読み込みに余分なステップを要するため、本当に必要な場合のみ複合構成にする

### Step 3: 配置

- **ユーザースコープ**: `~/.claude/skills/<skill-name>/SKILL.md`
- **プロジェクトスコープ**: `.claude/skills/<skill-name>/SKILL.md`

### Step 4: 案内

作成後、以下を伝える：
- `/<skill-name>` で手動起動できること
- 自動起動のトリガー条件
- **新しい会話で反映されること**（現在の会話では既にスキル一覧がロード済みのため）

### Step 5: セルフチェック

配置後、作成したスキルが `references/skill-design-guide.md` のプラクティスに沿っているか以下を確認する。問題があれば修正してからユーザーに案内する：

- [ ] **description**: 1024文字以内か。具体的なトリガーフレーズを複数含んでいるか。DO NOT TRIGGERで誤起動を防止しているか
- [ ] **本文行数**: 500行以内か。超過していればreferences/に分離する
- [ ] **Progressive Disclosure**: description / SKILL.md本文 / references/ の3層に情報が適切に配置されているか
- [ ] **命令形+理由**: 各手順が命令形で記述され、なぜそうするかの補足があるか
- [ ] **完了条件**: スキルのゴールが曖昧でなく、具体的に定義されているか
- [ ] **アンチパターン**: 肥大化・曖昧なdescription・過剰な権限に該当していないか

### 完了条件

SKILL.mdが所定のディレクトリに配置され、Step 5のセルフチェックを通過し、Step 4の案内をユーザーに伝えた時点で完了とする。
